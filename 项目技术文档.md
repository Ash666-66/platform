# 校园二手交易平台技术文档

## 1. 项目概述

校园二手交易平台是一个专为在校学生设计的二手物品交易系统，旨在方便学生之间的物品交易，减少资源浪费，促进校园内的物品流通。

### 1.1 项目目标

- 提供便捷的二手物品发布和浏览功能
- 实现安全可靠的交易流程
- 建立用户之间的信任机制
- 提供消息通知和评价系统

### 1.2 技术栈选择

- **前端**：Vue 2.6 + Element UI + Axios
- **后端**：Spring Boot + MyBatis-Plus + MySQL
- **认证**：JWT
- **部署**：前后端分离部署

## 2. 系统架构

### 2.1 系统层次结构

- **前端层**：Vue 组件、路由、状态管理
- **API层**：RESTful API 接口
- **服务层**：业务逻辑处理
- **数据访问层**：MyBatis-Plus 数据访问
- **数据层**：MySQL 数据库

### 2.2 核心模块

1. **用户管理模块**：注册、登录、个人信息管理
2. **商品管理模块**：发布、编辑、删除、查询商品
3. **交易管理模块**：创建交易、更新交易状态、查询交易
4. **消息通知模块**：系统消息、交易消息、聊天消息
5. **评价系统模块**：商品评价、用户评价

## 4. 核心功能实现

### 4.1 用户管理

#### 4.1.1 注册功能
- 前端：`Register.vue`
- 后端：`UserController.java`
- 流程：验证用户名和邮箱是否已存在 → 密码加密 → 保存用户信息

#### 4.1.2 登录功能
- 前端：`Login.vue`
- 后端：`UserController.java`
- 流程：验证用户名和密码 → 生成JWT令牌 → 返回用户信息

### 4.2 商品管理

#### 4.2.1 发布商品
- 前端：`ProductPublish.vue`
- 后端：`ProductController.java`
- 流程：验证用户登录 → 上传图片 → 保存商品信息

#### 4.2.2 商品列表
- 前端：`ProductList.vue`
- 后端：`ProductController.java`
- 流程：查询商品列表 → 支持分类筛选和关键词搜索

#### 4.2.3 商品详情
- 前端：`ProductDetail.vue`
- 后端：`ProductController.java`
- 流程：查询商品详情 → 显示卖家信息 → 支持购买操作

### 4.3 交易管理

#### 4.3.1 创建交易
- 前端：`ProductDetail.vue`
- 后端：`TransactionController.java`
- 流程：验证用户登录 → 检查商品状态 → 创建交易记录 → 发送消息通知

#### 4.3.2 交易状态更新
- 前端：`BuyerTransactions.vue`, `SellerTransactions.vue`
- 后端：`TransactionController.java`
- 流程：验证用户权限 → 更新交易状态 → 发送消息通知

### 4.4 消息通知

#### 4.4.1 消息列表
- 前端：`MessageList.vue`
- 后端：`MessageController.java`
- 流程：查询用户消息 → 支持标记已读

#### 4.4.2 消息详情
- 前端：`MessageDetail.vue`
- 后端：`MessageController.java`
- 流程：查询消息详情 → 支持标记已读

### 4.5 评价系统

#### 4.5.1 添加评价
- 前端：`ReviewAdd.vue`
- 后端：`ReviewController.java`
- 流程：验证用户登录 → 检查交易状态 → 保存评价信息

#### 4.5.2 评价查询
- 前端：`ProductDetail.vue`
- 后端：`ReviewController.java`
- 流程：查询商品评价 → 显示评价列表和平均评分

## 5. 系统安全

### 5.1 密码安全
- 使用 BCrypt 对密码进行加密存储
- 禁止明文传输密码

### 5.2 认证授权
- 使用 JWT 进行身份认证
- 实现基于角色的权限控制

### 5.3 输入验证
- 前端和后端双重验证
- 防止 SQL 注入和 XSS 攻击

### 5.4 数据安全
- 敏感数据加密存储
- 定期备份数据库

## 6. 部署与维护

### 6.1 开发环境
- 前端：Node.js + Vue CLI
- 后端：JDK 1.8 + Maven + Spring Boot
- 数据库：MySQL 5.7+

### 6.2 生产环境
- 前端：Nginx 静态资源服务
- 后端：Tomcat 或内嵌 Tomcat
- 数据库：MySQL 5.7+

### 6.3 维护建议
- 定期备份数据库
- 监控系统性能
- 及时更新依赖库
- 定期安全扫描

## 7. 技术难点与解决方案

### 7.1 图片上传
- 解决方案：使用文件服务器存储图片，数据库存储图片URL

### 7.2 交易状态管理
- 解决方案：使用状态机模式管理交易状态，确保状态转换的一致性

### 7.3 消息通知
- 解决方案：使用消息队列实现异步消息处理，提高系统响应速度

### 7.4 性能优化
- 解决方案：使用缓存机制减少数据库查询，优化SQL语句
